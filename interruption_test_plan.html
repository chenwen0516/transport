语音打断检测 · 全链路时延 · 误打断过滤
核心指标定义
打断延迟（用户感知）
T_tts_stop − T_user_speech_start
从用户开口到TTS停止播放的全程时间，是用户真实感受到的响应速度
系统响应延迟
T_vad_triggered − T_user_speech_start
VAD检测到语音起始的延迟，反映检测层性能
TTS停止延迟
T_tts_stop − T_vad_triggered
VAD触发后系统终止TTS播放的执行时间
误打断率
误触发次数 / 总测试次数
噪音、背景音等非语音输入导致TTS被错误中断的比率
打断延迟目标（统一）
< 500ms
任意时机打断均适用，不因早/中/晚而放宽
打断成功率
> 90%
所有打断场景平均，按会话计算
误打断率
< 5%
噪音场景下TTS被错误中断的比率

自动化实施代码框架
# interruption_test.py — 打断测试自动化框架 import asyncio, time, numpy as np, soundfile as sf class EventCapture: def __init__(self): self.events = [] self._t0 = time.monotonic() def record(self, name, payload=None): # 统一单调时钟，避免各组件时钟不同步 ts = (time.monotonic() - self._t0) * 1000 self.events.append({"name": name, "ts": ts, "data": payload}) def latency(self, from_event, to_event): t = {e["name"]: e["ts"] for e in self.events} return t[to_event] - t[from_event] class InterruptionTestCase: def __init__(self, id, delay_ms, user_audio, max_latency_ms=500): self.id = id self.interrupt_delay = delay_ms # 距TTS开始的打断时机 self.user_audio = user_audio self.max_latency = max_latency_ms async def run_case(ws_client, case, capture): # 1. 触发TTS播报 await ws_client.trigger_tts("请告诉我您的症状...") capture.record("tts_start") # 2. 等待指定时机再注入用户语音 await asyncio.sleep(case.interrupt_delay / 1000) capture.record("user_speech_start") await ws_client.send_audio(case.user_audio) # 3. 监听TTS停止事件 await ws_client.wait_for_event("tts_stopped", timeout=2.0) capture.record("tts_stop") # 4. 计算用户感知延迟（关键指标） latency = capture.latency("user_speech_start", "tts_stop") return { "case_id": case.id, "interruption_latency_ms": round(latency, 1), "passed": latency < case.max_latency }
实施前置条件检查
  • ✓
    已确认所有时间戳使用同一单调时钟源（time.monotonic），禁止混用系统时间
  • ✓
    已准备真实人声录音（至少50条），覆盖老年人、儿童、不同口音，禁止只用合成音频
  • ✓
    AEC（回声消除）测试（INT-010）必须在真实扬声器环境下执行，不可纯软件模拟
  • ✓
    prefix_padding_ms 建议设为200-300ms，并有专项用例验证首字不丢失（INT-015）
  • ✓
    打断延迟目标统一为500ms，早/中/晚打断不分级放宽
  • ✓
    每个用例至少运行30次，取P50/P90/P99，不以单次结果判断通过
  • ✓
    CI集成：每次合并前自动运行核心用例（INT-001～003, 008～010），输出与上版本的对比

轮次定义（精确版）
用户开口
T_speech_start
VAD触发
T_vad_start
静音超时
T_silence_end
轮次结束
T_turn_end
ASR完成
T_asr_done
LLM响应
T_llm_start
1轮 = 从用户开口（T_speech_start）到LLM开始回复（T_llm_start）的完整闭环。停顿期间未触发新speech_started事件，则属于同一轮次内部停顿，不切轮。
核心指标定义
VAD检测延迟
T_vad_start − T_speech_start
从用户真正开口到系统检测到语音的时间。需用注入时间戳作为T_speech_start，而非VAD自报时间
轮次结束延迟
T_turn_end − T_last_voice_frame
最后一帧语音到系统判定轮次结束的时间，等于silence_duration_ms ± 检测误差
字符错误率（CER）
编辑距离(ASR结果, 标注文本) / 标注文本长度
比"有无ASR结果"更严格，必须验证转写内容是否完整准确，尤其关注首字是否丢失
轮次划分准确率
边界误差 < 200ms的轮次数 / 总应划分轮次数
以200ms为容差窗口判断切轮时机是否正确，按轮次计数而非按字符

自动化实施代码框架
# turn_detection_test.py — 轮次检测自动化测试框架 import asyncio, jiwer, time class TurnTestCase: def __init__(self, id, audio_file, reference_text, expect_turn=True, pause_segments=None): self.id = id self.audio_file = audio_file self.reference = reference_text # 标注文本，用于算CER self.expect_turn = expect_turn # 是否期望触发轮次 self.pause_segments = pause_segments or [] # [(start_ms, duration_ms)] async def run_turn_case(ws_client, case, capture): # 注入音频，记录精确注入时间 capture.record("audio_inject") await ws_client.send_audio(case.audio_file) # 等待事件（最长等5秒） events_received = await ws_client.collect_events(timeout=5.0, events=[ "speech_started", "speech_ended", "transcription.completed", "response.created" ]) turn_triggered = "speech_started" in events_received asr_text = events_received.get("transcription.completed", {}).get("text", "") # 计算CER（字符错误率） cer = jiwer.cer(case.reference, asr_text) if asr_text else 1.0 # 计算VAD检测延迟 vad_latency = None if turn_triggered: vad_latency = (events_received["speech_started"]["ts"] - capture.get_ts("audio_inject")) return { "case_id": case.id, "turn_triggered": turn_triggered, "expected_turn": case.expect_turn, "vad_latency_ms": round(vad_latency, 1) if vad_latency else None, "asr_cer": round(cer, 3), "asr_text": asr_text, "passed": (turn_triggered == case.expect_turn) and (cer < 0.1) } # 中文停顿测试 — 动态构造停顿音频 def build_pause_audio(segments, pause_ms, sample_rate=16000): import numpy as np silence = np.zeros(int(sample_rate * pause_ms / 1000)) return np.concatenate([seg for pair in zip(segments, [silence] * len(segments)) for seg in pair])
实施前置条件与注意事项
  • ✓
    prefix_padding_ms 必须从0改为200ms，并用TRN-015验证首字不丢失，这是健康场景高风险缺陷
  • ✓
    TRN-006（600ms停顿切轮）需团队明确策略再定预期结果，不可跳过
  • ✓
    TRN-009（电视人声）是最难场景，建议单独统计，阈值调整时优先保证该项
  • ✓
    ASR评测必须使用jiwer等工具计算CER，禁止人工判断"大概对了"
  • ✓
    建立包含100+条真实录音的黄金数据集，每次版本更新自动跑CER对比，输出趋势图
  • ✓
    VAD参数调整必须联动测试：调threshold时同步跑TRN-001~003和TRN-008~009，防止顾此失彼
  • ✓
    TRN-017多轮累积测试，每轮延迟数据单独记录，画折线图观察是否有退化趋势

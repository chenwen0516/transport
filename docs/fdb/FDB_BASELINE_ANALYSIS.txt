二、总体结论
------------
Agent 已经可以稳定完成英文 FDB 全流程。基础轮次获取、停顿安全性，以及用户
打断后的回答质量较强，但还不能视为成熟的全双工系统，主要原因是：

1. 正常轮次和打断后的响应延迟接近 5 秒。
2. 背景讲话和用户对其他人讲话仍会频繁触发 Agent。
3. 用户 backchannel 的处理还不够稳定。
4. 当前 MMI 只运行在 Shadow 模式，本次分数不代表 MMI 带来的实际增益。

四、结果解释
------------
停顿安全性优秀，系统基本不会因用户短暂停顿而抢话。但约 5 秒的响应延迟说明
当前端点检测或后续链路偏保守，交互体验仍不自然。下一步需要把总延迟拆成：

用户结束 -> ASR final -> MMI 决策 -> LLM 首 token -> TTS 首音频

最严重的问题仍然是非目标讲话。背景讲话有 74% 会触发响应，用户对其他人讲话
也有 79% 会触发响应。这主要指向说话人归属、对话对象识别和 speaker gating，
单纯修改 prompt 很难可靠解决。

user_backchannel 的错误新响应率为 4.08%，仍高于 MMI 设计目标 3%；另外有
16.33% 被标为 C_UNCERTAIN_HANDLING，说明行为还不稳定。强打断正确响应率为
85.00%，低于设计目标 95%。因此目前不应直接把 MMI 切换到 Active。

停顿、正常轮次、用户打断、用户 backchannel 四个维度的派生宏平均约为
88.78%。该数字没有包含最困难的 background_speech 和 talking_to_other，
不能作为上线标准。
